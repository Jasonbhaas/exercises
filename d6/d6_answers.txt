What does git init do?
	- git init initializes a new git file. Say I'm working on some program. If I want git to start keeping track of my changes and updates, then I begin doing this with git init. From here on out, it tracks when I changed the file, and I can add/submit new versions ect.
What does git add do? How is it different from git commit?
	- git add stages new versions to commit but does not actually record/save the changes. Git Commit saves the new verison of the document and takes note of the changes between the versions. Commit is a little more permanent than add.
What is the difference between staged and committed (or versioned) files?
	- staged versions are not actually saved, so later on, you cannot go back and access them whereas you can with committed versions of a file.
What's the shortcut for adding all files and commiting them?
	- git add --all

What's the difference between GitHub and git? Could git exist without GitHub? Could GitHub exist without git?
	- Github is a site to publish all of your code on, and to back it up online. Git is the thing that manages all of your different versions of a document, and sends it to GitHub. Git could exist withough GitHub, but GitHub could not exist without Git.
